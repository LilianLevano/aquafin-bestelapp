FloodForecastService.php : 18

Generates, persists, and retrieves multi-year flood risk predictions for a given site.

Predictions are based on processed daily weather records from RiskCalculationService and are extrapolated over a 5-year horizon using seasonal precipitation multipliers derived from Belgian climate patterns. Results are stored in the FloodRiskAnalysis and RiskMonth tables via upsert (updateOrCreate) to avoid duplicate records across cache refreshes.

generateFiveYearPrediction()

FloodForecastService.php : 61

Generate a 5-year monthly flood risk prediction for the given site.

public generateFiveYearPrediction(array<string|int, mixed> $dailyRecords, int $siteId) : array<string|int, mixed>

Aggregates the provided daily records into monthly summaries via aggregateToMonthly(), then projects each month forward across 5 years by applying the seasonal multiplier for that month's season. If no real data exists for a given month, simulateBaseForMonth() is used to derive a baseline from Belgian climate averages. A month is flagged as extreme if its projected max risk >= 90 or total precipitation >= 100 mm.

Parameters
$dailyRecords : array<string|int, mixed>

Processed daily records from RiskCalculationService::processDailyRecords(). Each entry must contain: 'date' (date string), 'riskValue' (float|null), 'rainMm' (float|null), and 'humidity' (float|null).

$siteId : int

Primary key of the site to generate predictions for.

Return values

Flat array of monthly prediction entries, each containing: 'site_id', 'year', 'month', 'min_risk', 'max_risk', 'avg_risk', 'total_precipitation', 'avg_humidity', 'season', 'is_extreme'.

getCachedPredictions()

FloodForecastService.php : 180

Retrieve persisted predictions for a site from the database.

public getCachedPredictions(int $siteId) : array{analyses: array, riskMonths: array}|null

Returns all FloodRiskAnalysis and RiskMonth records for the given site from the current year onwards, sorted by year and month descending. Returns null if no analyses exist for the site.

Note: this method fetches all rows from both tables into memory before filtering. For large datasets, consider replacing ::all()->whereIn() with a query builder approach (e.g. FloodRiskAnalysis::where('site_id', $siteId)->where(...)->get()).

FloodForecastService.php : 117

Persist prediction entries to the database and identify risk months.

public savePredictions(array<string|int, mixed> $predictions, int $siteId) : RiskMonth[]}

Each prediction is upserted into the FloodRiskAnalysis table using (site_id, year, month) as the unique key, so re-running predictions for the same site does not produce duplicate records. If a prediction's avg_risk meets or exceeds RISK_THRESHOLD, a corresponding RiskMonth record is also upserted with the computed risk reason.

Aggregate daily weather records into per-month summaries.

private aggregateToMonthly(array<string|int, mixed> $dailyRecords) : array<int, array<string|int, mixed>>

Groups records by calendar month (1–12), collecting risk values, precipitation totals, and humidity values. Null values in any field are silently skipped. The resulting summary per month contains: 'min_risk', 'max_risk', 'avg_risk', 'total_precipitation', 'avg_humidity'.

Monthly summaries keyed by month number (1–12).

simulateBaseForMonth()

FloodForecastService.php : 275

Simulate a monthly baseline using Belgian climate averages.

private simulateBaseForMonth(int $month) : array<string|int, mixed>

Used as a fallback when no real forecast data is available for a given month. Precipitation and humidity values are approximate historical averages for Belgium. The risk score is derived using the same formula as RiskCalculationService: rain chance (capped at 100) weighted at 50%, humidity weighted at 30%.

Parameters
$month : int

Calendar month number (1–12).

Return values
array<string|int, mixed> —

Simulated baseline containing: 'min_risk', 'max_risk', 'avg_risk', 'total_precipitation' (mm), 'avg_humidity' (%).
